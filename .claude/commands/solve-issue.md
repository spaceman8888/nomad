description: GitHub 이슈 해결 플래너
---

GitHub 이슈를 분석하고 해결 계획을 수립합니다.

## 이슈 번호
$ARGUMENTS

## 작업 절차

### 1단계: 이슈 가져오기
```bash
gh issue view $ARGUMENTS
```
이슈의 제목, 본문, 라벨, 댓글 등 모든 정보를 확인합니다.

### 2단계: 코드베이스 분석
이슈와 관련된 코드를 탐색합니다:
- 관련 파일 및 함수 식별
- 현재 구현 방식 파악
- 영향 받는 범위 분석

### 3단계: 해결 계획 수립
다음 형식으로 상세 계획을 작성합니다:

```
## 이슈 요약
[이슈 내용 요약]

## 분석 결과
- 원인/요구사항 분석
- 관련 파일 목록

## 구현 계획
1. [첫 번째 작업]
   - 파일:
   - 변경 내용:
2. [두 번째 작업]
   ...

## 예상 영향
- 변경으로 인해 영향받는 부분

## 테스트 계획
- 검증 방법
```

### 4단계: 사용자 검토 요청
계획을 사용자에게 제시하고 AskUserQuestion 도구를 사용하여 다음을 확인합니다:
- 계획 승인 여부
- 수정이 필요한 부분
- 추가 고려사항

## 주의사항
- 사용자 승인 전까지 코드 수정을 하지 않습니다.
- 계획이 승인되면 구현을 시작합니다.
- 불명확한 부분은 반드시 사용자에게 질문합니다.
