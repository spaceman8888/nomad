# 노마드코리아 웹사이트 개선 명세서

## 프로젝트 개요
노마드코리아 웹사이트를 단계별로 개선하는 계획입니다. 각 Phase는 독립적으로 실행 가능하며, 데이터베이스 없이 가짜 데이터로 진행합니다.

---

## - [ ] Phase 1: 불필요한 페이지 및 네비게이션 정리

### 개선 사항 오버뷰
홈페이지와 인증 관련 페이지만 유지하고, 네비게이션에서 존재하지 않는 페이지로의 링크를 제거합니다. 사이트 구조를 단순화하여 핵심 기능에 집중합니다.

### 수정/개선해야 하는 기능
- [ ] `src/components/layout/Header.tsx` - 도시, 랭킹, 커뮤니티, 밋업 링크 삭제
- [ ] `src/components/layout/MobileNav.tsx` - 도시, 랭킹, 커뮤니티, 밋업 링크 삭제
- [ ] `src/components/layout/Footer.tsx` - 서비스/정보 섹션의 불필요한 링크 삭제
- [ ] `src/components/home/HeroSection.tsx` - 불필요한 페이지로의 링크가 있다면 삭제
- [ ] 홈페이지에서 `NomadLocations`, `RecentReviews`, `UpcomingMeetups` 섹션 삭제 (밋업/커뮤니티 관련)

### 작업 완료 후 검증/확인 사항
- [ ] 헤더에 로고와 인증 버튼(로그인/프로필)만 표시되는지 확인
- [ ] 모바일 네비게이션에 인증 관련 메뉴만 표시되는지 확인
- [ ] 푸터에 불필요한 링크가 없는지 확인
- [ ] 홈페이지에서 `HeroSection`, `QuickFilterTabs`, `FilterSortBar`, `CityCardGrid`만 표시되는지 확인
- [ ] 로그인/회원가입 페이지가 정상 작동하는지 확인
- [ ] 404 에러 없이 모든 링크가 정상 작동하는지 확인

### 관련 파일

## - [ ] Phase 2: 데이터 구조 및 필터 시스템 변경

### 개선 사항 오버뷰
기존 필터 시스템을 새로운 요구사항에 맞게 변경합니다. 예산, 지역, 환경, 최고 계절 4가지 필터를 적용하고 관련 데이터 구조를 수정합니다.

### 새로운 필터 정의
```typescript
// 예산
type Budget = '100만원 이하' | '100~200만원' | '200만원 이상';

// 지역
type Region = '전체' | '수도권' | '경상도' | '전라도' | '강원도' | '제주도' | '충청도';

// 환경
type Environment = '자연친화' | '도심선호' | '카페작업' | '코워킹 필수';

// 최고 계절
type BestSeason = '봄' | '여름' | '가을' | '겨울';
```

### 수정/개선해야 하는 기능
- [ ] `src/data/mock-data.ts` - City 인터페이스에 새 필드 추가 (budget, region, environment, bestSeason)
- [ ] `src/data/mock-data.ts` - 기존 도시 데이터에서 불필요한 필드 제거 (nomadScore, internetSpeed, recommendRate, safetyGrade, weather, aqi, currentNomads)
- [ ] `src/components/home/QuickFilterTabs.tsx` - 새 필터 탭으로 변경 (예산별, 지역별, 환경별, 계절별)
- [ ] `src/components/home/FilterSortBar.tsx` - 새 필터 드롭다운으로 변경
- [ ] 필터 타입 정의 파일 생성 (`src/types/filters.ts`)

### 작업 완료 후 검증/확인 사항
- [ ] 새로운 필터 타입이 정상적으로 정의되어 있는지 확인
- [ ] QuickFilterTabs에 예산/지역/환경/계절 필터가 표시되는지 확인
- [ ] FilterSortBar에 새로운 필터 드롭다운이 작동하는지 확인
- [ ] mock-data의 City 인터페이스가 새 구조를 반영하는지 확인
- [ ] TypeScript 컴파일 에러가 없는지 확인

### 관련 파일
- `src/data/mock-data.ts`
- `src/components/home/QuickFilterTabs.tsx`
- `src/components/home/FilterSortBar.tsx`
- `src/types/filters.ts` (신규 생성)

## - [ ] Phase 3: 카드 구조 변경

### 개선 사항 오버뷰
도시 카드의 정보 표시 방식을 변경합니다. 기존 노마드 점수, 월 생활비, 인터넷 속도, 추천율 대신 예산, 지역, 환경, 최고 계절을 Key-Value 형태로 표시합니다.

### 수정/개선해야 하는 기능
- [ ] `src/components/home/CityCard.tsx` - 카드 콘텐츠 영역 재구성
  - [ ] 기존 4개 지표 (노마드 점수, 월 생활비, 인터넷 속도, 추천율) 삭제
  - [ ] 새로운 Key-Value 레이아웃 구현 (예산, 지역, 환경, 최고 계절)
- [ ] `src/components/home/CityCard.tsx` - 푸터 영역 정리 (날씨, AQI, 안전등급 삭제)
- [ ] `src/components/home/CityCard.tsx` - 좌상단 배지 영역 정리 (순위, HOT 배지 유지 또는 삭제 결정)
- [ ] `src/components/home/CityCardGrid.tsx` - 새 데이터 구조에 맞게 props 전달 수정

### 카드 새 레이아웃 예시
```
┌─────────────────────────┐
│ [제주시 배경 이미지]     │
│ #1 HOT                  │

│ 제주도                   │
│                         │
│ 예산      100만원 이하   │
```

## - [ ] Phase 4: 좋아요/싫어요 시스템 구현

### 개선 사항 오버뷰
기존 별점 평점 시스템을 좋아요/싫어요 버튼 시스템으로 변경합니다. 버튼 클릭 시 상태에 따라 아이콘 색상이 변경되고, 좋아요/싫어요 숫자가 업데이트됩니다.

### 수정/개선해야 하는 기능
- [ ] `src/data/mock-data.ts` - City 인터페이스에 likes, dislikes 필드 추가
- [ ] `src/data/mock-data.ts` - 각 도시 데이터에 likes, dislikes 가짜 숫자 추가
- [ ] `src/components/home/CityCard.tsx` - 좋아요/싫어요 버튼 UI 추가
  - [ ] 좋아요 버튼 (👍 아이콘 + 숫자)
  - [ ] 싫어요 버튼 (👎 아이콘 + 숫자)
- [ ] 좋아요/싫어요 상태 관리 로직 구현 (React useState 사용)
  - [ ] 좋아요 클릭 시: 좋아요 활성화, 싫어요 비활성화
  - [ ] 싫어요 클릭 시: 싫어요 활성화, 좋아요 비활성화
  - [ ] 같은 버튼 재클릭 시: 해당 상태 비활성화
- [ ] 버튼 상태에 따른 스타일 적용
  - [ ] 비활성: 회색 아이콘
  - [ ] 좋아요 활성: 파란색/초록색 아이콘
  - [ ] 싫어요 활성: 빨간색 아이콘

### 상태 흐름
```
초기 상태: liked=false, disliked=false
좋아요 클릭 → liked=true, disliked=false, likes+1
싫어요 클릭 → liked=false, disliked=true, likes-1, dislikes+1
좋아요 재클릭 → liked=false, disliked=false, likes-1
```

### 작업 완료 후 검증/확인 사항
- [ ] 좋아요 버튼 클릭 시 아이콘 색상이 변경되는지 확인
- [ ] 싫어요 버튼 클릭 시 아이콘 색상이 변경되는지 확인
- [ ] 좋아요 클릭 시 좋아요 숫자가 증가하는지 확인
- [ ] 싫어요 클릭 시 싫어요 숫자가 증가하는지 확인
- [ ] 좋아요 상태에서 싫어요 클릭 시 좋아요가 해제되고 싫어요가 활성화되는지 확인
- [ ] 같은 버튼 재클릭 시 상태가 해제되는지 확인
- [ ] 별점 평점이 완전히 제거되었는지 확인

### 관련 파일
- `src/components/home/CityCard.tsx`
- `src/data/mock-data.ts`

## - [ ] Phase 5: 가짜 데이터 완성

### 개선 사항 오버뷰
모든 도시 데이터를 새로운 필터 시스템에 맞게 업데이트합니다. 각 필터 옵션(예산, 지역, 환경, 최고 계절)에 해당하는 도시가 최소 하나씩 포함되도록 데이터를 구성합니다.

### 필터 옵션별 최소 도시 배치 계획

| 필터 | 옵션 | 최소 도시 예시 |
|------|------|---------------|
| 예산 | 100만원 이하 | 전주, 강릉 |
| 예산 | 100~200만원 | 부산, 대전 |
| 예산 | 200만원 이상 | 서울, 제주시 |
| 지역 | 수도권 | 서울 |
| 지역 | 경상도 | 부산 |
| 지역 | 전라도 | 전주 |
| 지역 | 강원도 | 강릉 |
| 지역 | 제주도 | 제주시 |
| 지역 | 충청도 | 대전 |
| 환경 | 자연친화 | 제주시, 강릉 |
| 환경 | 도심선호 | 서울, 부산 |
| 환경 | 카페작업 | 전주 |
| 환경 | 코워킹 필수 | 서울 |
| 최고 계절 | 봄 | 전주 |
| 최고 계절 | 여름 | 강릉, 부산 |
| 최고 계절 | 가을 | 서울 |
| 최고 계절 | 겨울 | 제주시 |

### 수정/개선해야 하는 기능
- [ ] `src/data/mock-data.ts` - 기존 6개 도시 데이터 업데이트
  - [ ] 제주시: 200만원 이상, 제주도, 자연친화, 겨울
  - [ ] 부산: 100~200만원, 경상도, 도심선호, 여름
  - [ ] 서울: 200만원 이상, 수도권, 코워킹 필수, 가을
  - [ ] 강릉: 100만원 이하, 강원도, 자연친화, 여름
  - [ ] 전주: 100만원 이하, 전라도, 카페작업, 봄
  - [ ] 대전: 100~200만원, 충청도, 도심선호, 가을
- [ ] 각 도시에 likes, dislikes 숫자 추가
- [ ] 필요시 추가 도시 데이터 생성 (필터 옵션 커버리지 확보)

### 작업 완료 후 검증/확인 사항
- [ ] 모든 예산 옵션에 해당하는 도시가 있는지 확인
- [ ] 모든 지역 옵션에 해당하는 도시가 있는지 확인
- [ ] 모든 환경 옵션에 해당하는 도시가 있는지 확인
- [ ] 모든 최고 계절 옵션에 해당하는 도시가 있는지 확인
- [ ] 필터 선택 시 해당하는 도시가 올바르게 필터링되는지 확인
- [ ] 모든 도시 카드가 정상적으로 렌더링되는지 확인
